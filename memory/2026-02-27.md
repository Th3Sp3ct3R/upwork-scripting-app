**Reaction Types Supported (Python Service):**
- LIKE (post)
- UNLIKE

- COMMENT
- STORY_VIEW
- STORY_LIKE
- GET_USER_STORIES
- GET_USER_FEED
- GET_FOLLOWERS
- GET_FOLLOWING
- GET_TAG_FEED

**Backend Endpoints (Node.js):**
- POST /api/reactions/like-post
- POST /api/reactions/comment-post
- POST /api/reactions/view-story
- POST /api/reactions/watch-story
- POST /api/reactions/engage-post (like + comment)
- POST /api/reactions/batch
- GET /api/reactions/history

**Database Tables:**
- account_actions: Logs every like/comment/follow/view action
- action_analytics: Measures engagement per action

**Flow:**
1. Frontend calls POST /api/reactions/like-post (with auth)
2. Node.js ReactionExecutorService enforces delay (3-15s random)
3. Calls Python service at http://localhost:8000/action/like
4. Python service uses instagrapi to execute action
5. Response logged to account_actions table
6. Frontend can query GET /api/reactions/history

**Why User Asked to Check:**
Want to see REAL action logs from working reactions before building the Mission Control UI streaming layer. The backend logging already exists; need to verify what's actually in the logs when reactions execute.
